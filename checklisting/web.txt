## Functionality — input testing loop

> For EVERY input (params, headers, cookies, body fields, uploads) run through this:

- [ ] Reflection test → does input echo back? (→ XSS)
  - found:

- [ ] Interpretation test → `7*7` `{{7*7}}` `${7*7}` `<%= 7*7 %>` (→ SSTI)
  - found:

- [ ] Execution test → `;id` `|whoami` `` `id` `` (→ command injection)
  - found:

- [ ] File/path test → `../../etc/passwd` (→ LFI)
  - found:

- [ ] ID test → increment, swap, remove (→ IDOR)
  - found:

---

- [ ] File upload
  - bypass extension filters: double ext, null bytes, case, content-type swap
  - magic bytes if content is validated
  - where does the file go after upload? can you reach it?
  - notes:

- [ ] LFI/RFI — any param that loads a file or path
  - try `php://filter` `php://input` `data://` wrappers if PHP
  - log poisoning if you can write to a log (user-agent → `/var/log/apache2/access.log`)
  - `/proc/self/environ`, `/proc/self/cmdline` worth trying
  - params tested:

- [ ] IDOR — every ID-based param
  - try as unauthenticated, as different user, with different HTTP methods
  - GUIDs aren't safe either — if you can register two accounts, test cross-account
  - params tested:

- [ ] Authentication
  - default/weak creds, credential stuffing with found usernames
  - password reset / forgot password flows — token predictability, host header injection
  - JWT: alg:none, weak secret, kid injection
  - type juggling if PHP (loose comparisons)
  - auth bypass: SQLi on login, empty password, array params
  - username enumeration via timing or different error messages
  - tried:

- [ ] SSRF — any param that fetches a URL or hostname
  - try: `127.0.0.1` `169.254.169.254` `10.x` `192.168.x`
  - bypass filters: `http://0.0.0.0` decimal IPs, `http://127.1`
  - params tested:

- [ ] API endpoints
  - try all methods: GET POST PUT PATCH DELETE OPTIONS
  - test with no auth header, different user's token
  - fuzz undocumented endpoints from JS files
  - check if v1 endpoint still exists when you're on v2
  - found:

- [ ] Verbose error messages → stack traces reveal paths, params, versions
  - trigger with: unexpected types, missing params, very long input, null bytes
  - found:

- [ ] Rate limiting — test login, password reset, OTP fields
  - if rate limited by IP try `X-Forwarded-For: 127.0.0.1`
  - notes:

## Conditionals — only if you see these

### if you see XML input
- [ ] XXE → inject external entity, try `file:///etc/passwd`
- [ ] blind XXE → OOB with external DTD
- notes:

### if you see a search / query field
- [ ] SQLi → `'` `"` `OR 1=1--` UNION SELECT, error-based, blind time-based
- [ ] NoSQLi → `{"$gt":""}` `[$ne]=` `{"$where":"1==1"}`
- [ ] LDAP injection if auth looks directory-backed
- notes:

### if you see a redirect / URL param
- [ ] open redirect → try combining with other flows (password reset, OAuth)
- notes:

### if you see a template / email / report generator
- [ ] SSTI deeper → `{{config}}` `{{self.__class__}}`, Jinja2/Twig/Freemarker payloads
- [ ] HTML injection → if rendered to PDF try `<script>` `<img src=x>`
- notes:

### if you see deserialization (Java, PHP, .NET)
- [ ] PHP: object injection via `O:4:"User":...`
- [ ] Java: ysoserial payloads — identify the library first
- [ ] check `Content-Type: application/x-java-serialized-object`
- notes:

### if you see WebSockets
- [ ] intercept in Burp → same injection tests as HTTP params
- [ ] check if auth is enforced per-message or just on handshake

## Things people miss

- [ ] HTML comments on error pages specifically, not just main pages
- [ ] Backup files: `index.php.bak` `index.php~` `config.php.old`
- [ ] Parameter pollution: `param=a&param=b` — which value does the app use?
- [ ] HTTP method override: `X-HTTP-Method-Override: PUT` in a POST request
- [ ] Dotfiles if you have file read: `.bashrc` `.bash_history` `.env` `.ssh/id_rsa`
- [ ] Source of every JS file, not just the main bundle
- [ ] Admin paths even if not linked: `/admin` `/manager` `/console` `/phpmyadmin` `/dashboard`
- [ ] Second-order injection — input stored then rendered elsewhere (profile page, logs, emails)
- [ ] Retest everything after you get auth — behavior changes a lot
- [ ] If you find creds → try them on every other service immediately
- [ ] Chain low-severity bugs — IDOR + upload, redirect + reset token, etc.
- [ ] 403 on a path doesn't mean dead — try appending `/` or `/.` or changing method
- [ ] Integer/type params: send a string, send negative, send 0, send very large number
- [ ] If the app has a "remember me" or persistent session — inspect that token separately
