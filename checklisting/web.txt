Web

Enumeration

Directory/file fuzzing (feroxbuster/ffuf) with a solid wordlist, include extensions (.php .txt .bak .old .html .js .config)


Fuzz for vhosts


Check robots.txt, sitemap.xml, .git, .env, .htaccess


View page source on every page — comments, hidden fields, hardcoded credentials


Check all JS files for endpoints, credentials, tokens


Look at all cookies — decode/inspect them


Check HTTP headers for anything unusual


_______________

Functionality

Test every input field for SQLi, XSS, command injection, SSTI


Test file upload — bypass extension filters, content-type, magic bytes


Test for LFI/RFI on any parameter that looks like it loads a file


Check for IDOR on any ID-based parameters


Test authentication — default creds, weak passwords, type juggling, JWT manipulation


Check for exposed API endpoints and how they handle unexpected input


Look for verbose error messages — stack traces often reveal code and parameters


____
Things people miss

HTML comments on every single page including error pages
Source code of every JS file
All dotfiles (.bashrc, .bash_profile, .env etc.) if you get file read
Backup files of known pages (index.php.bak, index.php~)
Parameter pollution and unexpected HTTP methods (PUT, DELETE, PATCH)
