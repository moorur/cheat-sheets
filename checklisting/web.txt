Functionality
🔹 Core input testing (replace your big list mentally with this loop)

For every input (params, headers, cookies, uploads):

Reflection test
Interpretation test → 7*7, {{7*7}}, ${7*7}
Execution test → ;id, |whoami
File/reference test → ../../etc/passwd, IDs

(then go deeper only if something reacts)

Test file upload
→ bypass extension filters, content-type, magic bytes
→ try double extensions, null bytes, case variations

Test for LFI/RFI
→ any parameter that loads files
→ try wrappers (php://filter, etc.) if applicable

Check for IDOR
→ any ID-based parameters
→ try incrementing, swapping, accessing as other users

Test authentication
→ default creds, weak passwords
→ type juggling, JWT manipulation
→ password reset / forgot password flows

Check for exposed API endpoints
→ how they handle unexpected input
→ test with different methods (GET/POST/PUT/PATCH/DELETE)

Look for verbose error messages
→ stack traces reveal code/paths/params
